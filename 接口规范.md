# 一、认证模块补充

## 1.1 学生注册接口

说明：

本接口仅用于学生用户注册。

商家账号不通过注册接口创建，而是通过：

```http
POST /api/merchant/apply
```

提交入驻申请。

管理员账号由系统后台初始化或超级管理员创建。

---

## 接口地址

```http
POST /api/auth/register/student
```

---

## 请求参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| username | String | 是 | 用户名 |
| password | String | 是 | 登录密码 |
| realName | String | 是 | 真实姓名 |
| phone | String | 是 | 手机号 |

---

## 请求示例

```json
{
  "username": "zhangsan",
  "password": "123456",
  "realName": "张三",
  "phone": "13800000000"
}
```

# 二、消息模块补充

---

# 2.1 学生联系商家

## 接口地址

```http
POST /api/student/message/send
```

---

## 功能说明

学生针对某订单联系商家。

---

## 请求参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| orderId | Long | 是 | 订单ID |
| merchantId | Long | 是 | 商家ID |
| content | String | 是 | 消息内容 |

---

## 请求示例

```json
{
  "orderId": 1001,
  "merchantId": 1,
  "content": "请问多久可以做好？"
}
```

---

# 2.2 学生联系客服

## 接口地址

```http
POST /api/student/message/to_customer
```

---

## 功能说明

学生联系平台客服。

---

## 请求参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| orderId | Long | 否 | 订单ID |
| type | String | 是 | 问题类型 |
| content | String | 是 | 消息内容 |

---

## type 可选值

| 值 | 含义 |
|---|---|
| order | 订单问题 |
| payment | 支付问题 |
| complaint | 投诉建议 |
| other | 其他 |

---

## 请求示例

```json
{
  "orderId": 1001,
  "type": "payment",
  "content": "支付成功但订单未更新"
}
```

# 3.1 修改商家信息

## 接口地址

```http
PUT /api/merchant/info
```

---

## 功能说明

商家修改店铺基础信息。

- 店铺简介
- 联系电话
- 店铺Logo
- 店铺地址

等信息。

---

## 请求参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| shopName | String | 否 | 店铺名称 |
| phone | String | 否 | 联系电话 |
| address | String | 否 | 店铺地址 |
| logo | String | 否 | Logo图片URL |
| description | String | 否 | 店铺简介 |

---

## 请求示例

```json
{
  "phone": "13811112222",
  "description": "主营轻食与低脂套餐",
  "logo": "https://xxx.com/logo.png"
}
```

# 七、接口设计补充说明

---

# 7.1 关于 PUT 与 PATCH

当前系统统一使用：

```http
PUT
```

进行状态更新操作。

例如：

```http
PUT /api/student/orders/{id}/cancel
```

虽然：

```http
PATCH
```

更符合“部分更新”的 RESTful 语义，但为了：

- 降低前后端学习成本
- 保持团队接口风格统一
- 简化 SpringBoot Controller 开发

本项目统一采用：

```http
PUT
```

实现状态修改类接口。
