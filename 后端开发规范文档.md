### 1.1 分层职责说明

| 层级 | 目录 | 职责 |
|------|------|------|
| 控制器层 | `controller/` | 接收 HTTP 请求，参数校验，调用 Service，返回 Result |
| 业务层 | `service/` + `service/impl/` | 核心业务逻辑，事务管理 |
| 持久层 | `mapper/` | MyBatis Plus Mapper 接口，数据库操作 |
| 实体层 | `entity/` | 数据库表映射实体类 |
| DTO 层 | `dto/` | 前端传入的数据传输对象 |
| VO 层 | `vo/` | 返回给前端的视图对象 |
| 公共层 | `common/` | 配置、拦截器、异常处理、统一返回 |
| 工具层 | `utils/` | 通用工具类 |

### 1.2 控制器按角色拆分

| 路径 | 目标角色 | 说明 |
|------|----------|------|
| `controller/auth/` | 公开 | 登录、注册（不需要 Token） |
| `controller/student/` | 学生 | 浏览菜品、下单、查看订单 |
| `controller/merchant/` | 商家 | 管理菜品、处理订单 |
| `controller/admin/` | 管理员 | 用户管理、系统配置 |

```yaml
server:

  datasource:

         &useSSL=false

    password: root                        # 生产环境需修改

  jackson:
    date-format: yyyy-MM-dd HH:mm:ss      # 全局日期格式
    time-zone: Asia/Shanghai
    serialization:
      write-dates-as-timestamps: false    # 不输出时间戳

mybatis-plus:
  type-aliases-package: com.campus.entity # 实体类扫描
  mapper-locations: classpath*:/mapper/**/*.xml
  configuration:
    map-underscore-to-camel-case: true    # 下划线转驼峰
    log-impl: org.apache.ibatis.logging.stdout.StdOutImpl
  global-config:
    db-config:
      id-type: assign_id                  # 雪花算法主键
      logic-delete-field: deleted         # 逻辑删除字段
      logic-delete-value: 1
      logic-not-delete-value: 0

logging:
  level:
    com.campus: debug                     # 项目包 DEBUG 级别
    com.baomidou: debug
    org.springframework: warn

jwt:
  secret: campus-dining-jwt-secret-key-2024-must-be-at-least-256-bits-long
  expiration: 604800000                   # 7 天（毫秒）

knife4j:
  enable: true
  setting:
    language: zh_cn
```

### 3.1 关键配置说明

| 配置项 | 说明 |
|--------|------|
| `id-type: assign_id` | 使用雪花算法生成分布式唯一 ID |
| `logic-delete-field: deleted` | 逻辑删除，删除操作变为 UPDATE deleted=1 |
| `map-underscore-to-camel-case: true` | 数据库字段 `user_name` 自动映射到 `userName` |
| `jwt.secret` | 生产环境必须使用环境变量或配置中心注入 |

## 四、统一返回规范

### 4.1 ResultCode 枚举

文件路径：[ResultCode.java](src/main/java/com/campus/common/result/ResultCode.java)

| 枚举值 | 状态码 | 说明 |
|--------|--------|------|
| `SUCCESS` | 200 | 操作成功 |
| `BAD_REQUEST` | 400 | 请求参数不正确 |
| `UNAUTHORIZED` | 401 | 未授权，Token 缺失或无效 |
| `FORBIDDEN` | 403 | 权限不足 |
| `NOT_FOUND` | 404 | 资源不存在 |
| `ERROR` | 500 | 服务器内部错误 |

### 4.2 Result\<T\> 统一返回类

文件路径：[Result.java](src/main/java/com/campus/common/result/Result.java)

**结构体**：

```json
{
  "code": 200,
  "message": "操作成功",
  "data": { }
}
```

**静态方法速查**：

| 方法 | 用途 | 示例 |
|------|------|------|
| `Result.success()` | 成功，无数据 | `return Result.success();` |
| `Result.success(data)` | 成功，带数据 | `return Result.success(user);` |
| `Result.success(msg, data)` | 成功，自定义消息 | `return Result.success("登录成功", token);` |
| `Result.error()` | 失败，默认 500 | `return Result.error();` |
| `Result.error(msg)` | 失败，自定义消息 | `return Result.error("用户名已存在");` |
| `Result.error(code, msg)` | 失败，自定义码+消息 | `return Result.error(400, "参数错误");` |
| `Result.error(ResultCode)` | 失败，使用枚举 | `return Result.error(ResultCode.FORBIDDEN);` |

### 4.3 开发规范

- **所有 Controller 方法必须返回 `Result<T>`**，不得直接返回实体或字符串
- 成功场景使用 `Result.success()` 系列方法
- 业务异常直接抛出 `RuntimeException`（或其子类），由全局异常处理器统一捕获
- 不要在 Controller 中手动 `try-catch` 后返回 `Result.error()`

## 五、异常处理规范

### 5.1 全局异常处理器

文件路径：[GlobalExceptionHandler.java](src/main/java/com/campus/common/exception/GlobalExceptionHandler.java)

**处理链路**：

| 异常类型 | HTTP 状态码 | 处理逻辑 |
|----------|-------------|----------|
| `MethodArgumentNotValidException` | 400 | 提取字段校验错误信息，拼接返回 |
| `RuntimeException` | 500 | 捕获所有业务异常，返回异常消息 |
| `Exception` | 500 | 兜底处理，记录日志，返回通用错误 |

### 5.2 使用建议

- **业务异常**：直接 `throw new RuntimeException("具体错误信息")`，前端会收到 `{"code":500,"message":"具体错误信息"}`
- **参数校验**：在 DTO 中使用 `@NotNull`、`@NotBlank` 等注解，校验失败自动返回 400 + 详细错误
- **自定义异常**：后续可扩展 `BusinessException` 继承 `RuntimeException`，携带自定义状态码

## 六、JWT 认证规范

### 6.1 JwtUtil 工具类

文件路径：[JwtUtil.java](src/main/java/com/campus/utils/JwtUtil.java)

**核心方法**：

| 方法 | 参数 | 返回值 | 说明 |
|------|------|--------|------|
| `generateToken(userId, role)` | 用户ID, 角色 | `String` | 生成 JWT Token |
| `parseToken(token)` | Token 字符串 | `Claims` / `null` | 解析 Token，失败返回 null |
| `getUserId(token)` | Token 字符串 | `Long` / `null` | 从 Token 提取用户 ID |
| `getRole(token)` | Token 字符串 | `String` / `null` | 从 Token 提取角色 |
| `isTokenValid(token)` | Token 字符串 | `boolean` | 校验 Token 是否有效 |

**Token 结构**：

```
Header:  { "alg": "HS256" }
Payload: { "sub": "1", "role": "student", "iat": ..., "exp": ... }
```

### 6.2 JwtInterceptor 拦截器

文件路径：[JwtInterceptor.java](src/main/java/com/campus/common/interceptor/JwtInterceptor.java)

**拦截流程**：

```
请求 → OPTIONS 预检? → 直接放行
     → 提取 Authorization 头
     → 是否以 "Bearer " 开头? → 否 → 返回 401
     → 解析 Token 是否有效? → 否 → 返回 401
     → 将 userId、role 写入 request attribute → 放行
```

### 6.3 WebMvcConfig 拦截器注册

文件路径：[WebMvcConfig.java](src/main/java/com/campus/common/config/WebMvcConfig.java)

**拦截规则**：

| 路径模式 | 是否需要 Token | 说明 |
|----------|---------------|------|
| `/api/auth/**` | 否 | 登录、注册接口放行 |
| `/doc.html` | 否 | Knife4j 文档页放行 |
| `/swagger-ui/**` | 否 | Swagger UI 资源放行 |
| `/v3/api-docs/**` | 否 | OpenAPI JSON 放行 |
| `/api/**`（其余） | 是 | 所有业务接口需携带 Token |

### 6.4 前端对接方式

**请求头**：

```
Authorization: Bearer <token>
```

**登录后获取 Token**（示例）：

```json
// POST /api/auth/login
// Response:
{
  "code": 200,
  "message": "登录成功",
  "data": "eyJhbGciOiJIUzI1NiJ9.eyJzdWIiOiIxIiwicm9sZSI6InN0dWRlbnQifQ..."
}
```

**后续请求携带 Token**：

```javascript
fetch('/api/student/orders', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});
```

## 十、开发规范

### 10.1 命名规范

| 类型 | 命名规则 | 示例 |
|------|----------|------|
| 实体类 | 表名驼峰 | `User`, `Dish`, `Order` |
| Mapper | 实体名 + Mapper | `UserMapper` |
| Service 接口 | 实体名 + Service | `UserService` |
| Service 实现 | 实体名 + ServiceImpl | `UserServiceImpl` |
| Controller | 模块名 + Controller | `StudentController` |
| DTO | 操作 + 实体 + DTO | `LoginDTO`, `PageQueryDTO` |
| VO | 实体 + VO | `DishVO`, `OrderVO` |

### 10.2 包结构规范

- 所有代码必须在 `com.campus` 包下
- Controller 按角色分包子目录
- Service/Mapper/Entity 不分子目录，通过类名前缀区分模块

### 10.3 代码规范

- 所有类必须使用 Lombok 注解（`@Data`、`@RequiredArgsConstructor`、`@Slf4j`）
- 日志使用 `@Slf4j` 提供的 `log` 对象，不使用 `System.out`
- 所有类、方法必须有中文注释
- 使用构造注入（`@RequiredArgsConstructor` + `final`），禁止字段注入（`@Autowired`）
- Controller 只做参数校验和调用 Service，不写业务逻辑

### 10.4 Git 提交规范

```
feat: 新功能     feat: 添加用户登录接口
fix:  修复 Bug   fix: 修复 Token 过期判断错误
docs: 文档变更   docs: 更新 API 文档
ref:  重构       ref: 统一异常处理重构
```

## 十一、后续开发指引

### 11.1 数据库建表后

1. 在 `entity/` 下创建实体类，使用 `@TableName` 指定表名
2. 在 `mapper/` 下创建 Mapper 接口，继承 `BaseMapper<Entity>`
3. 在 `service/` 下创建 Service 接口，继承 `IService<Entity>`
4. 在 `service/impl/` 下创建 Service 实现，继承 `ServiceImpl<Mapper, Entity>`
5. 在对应 Controller 包下创建控制器

### 11.2 添加新接口的检查清单

- [ ] DTO/VO 是否创建并添加了校验注解
- [ ] Controller 是否添加了 `@Tag` / `@Operation` 注解
- [ ] 接口路径是否符合 RESTful 规范
- [ ] 是否需要 Token 鉴权（不在 `/api/auth/**` 下的自动拦截）
- [ ] 返回值是否使用 `Result<T>` 封装
- [ ] 异常是否通过 `throw new RuntimeException()` 抛出
